Axeltov isn't executing anything to the CTP Listener Channel!? New

There are a few possible reasons why Axeltov isn't responding or executing the task to the channel. We'll split it to 3 parties; Discord, Axeltov Team and the End User (Server Owners/Administrators). On behalf of the Discord Party, their API system's may be laggy, sluggish and Axeltov couldn't pickup the information provided by Discord due to this. We can't really fix this problem, but only Discord themselves. As of the Axeltov Team, we may be preforming ongoing maintenance on the bot, which you can check out on our support page, updates channel or the website. If there are no ongoing maintenance or server outages, then that may be a possible bug with the bot which we can happily restart safely and quickly when needed. As of the End User, check if there's any spelling mistakes in the channel, Maybe the ID uses an l (L for Lima) instead of an I (I for India) which as you can see that they look similiar. If spelling isn't an issue and both parties don't have any ongoing issues as well, please contact the Support Team as they try their best to analyse the problem and solve it for you.

Why does the bot uses CTP Listeners

It allows customization of the Discord Server by telling the bot that this channel is used for this and for that. We plan to add more CTP Listeners in the future, so check around the modules if theres a New! banner next to it!

How can I suggest a CTP Listener idea? Updated

You can pop your suggestions over in the #suggestions channel in our Support Discord! We'd love to hear your idea and the community will also have a vote in on it to see if this suggestion is highly demanded!

Axeltov is incorrectly using a CTP Listener/Outputting wrong information through a different ID header?

It's more likely to be a bug which can be fixed quickly and efficiently. If one of the channels that has a set CTP that isn't working as it describes to be, make sure to remove the CTP from the channel topic, file and submit a bug report in our Support Discord and we'll happily look over it!

Axeltov - GAPI Partnership

What is the GAPI Partnership?

GAPI, Global API, is a Partnership Program which allows Axeltov Servers that fits the requirements, approved by RediPanda, or one of the Lead Support Team Members to completely utilise the GAPI commands and systems. These include Globalised Bans, Globalised Warnings, Globalised Profiles and usually receive perks earlier than the Public.

How do I participate in the GAPI Partnership?

To participate and get your server authorised to use GAPI Systems and Commands, you must fit the server requirements:

Discord Server Requiremnets:
											
    1) Must have more than or atleast 150~250+ online and ACTIVE users.
											
	2) RediPanda (Bot Author) must enter the server, inspects channels and server policies in regards to warns/bans.
		     (To see if the server uses this to abuse the system.)
	3) A temp role must be created, does not need to have special or additional permissions except access to the main staff chat and needs to be named "Axeltov Manager".
	         (Role Created so that Redi or an LSTM can read the staff channel without granting any additional moderation perms.)
	4) A permanent role must be created, named "AxelGStaff". This role allows the bot to grant the user who possess this role access to the GAPI
		     
	4) All Staff of the Discord Server must be notified of new powers granted to them. Redi or the LSTM has to witness these statements and confirmations from each assigned staff that has that granted trust-role.
	         (This is used as proof that we can use this evidence to withdraw your server from the GAPI Program if the server/staff breaks the TOS.)

Why should my server participate in the GAPI Partnership?

For starters, it's free ðŸ˜„. And secondly, you receive additional benefits that not only do you have access to our Moderation Database which is logged with other Partnership servers, you also gain additional early-access features which can be utilised to improve your server.
