Bot CTP Listeners

Some CTP Listeners May Be Bugged

Some listeners may not be functioning as normal due to the bot's update/query speed. If there's any problems setting up a CTP Listener, contact the support server.

What are Bot CTP Listeners?

Bot CTP Listeners (also known as Bot Channel Topic Protocols), are pre-assigned to every server, allowing more control and functionality on the server. By putting these CTP ID's into the Channel's Topic area, you are allowing the bot to use that channel depending on the type of ID you've place in there.

Examples:

Welcome Channel Welcome_Channel

Axeltov Updates Axeltov_Bot_Update

Lists of CTP Listeners

We plan to add more CTP Listeners in the future! Please bear with us! If you have any suggestions, please place it in the suggestion channel in our server!

Don't put 2 CTP Listeners in the same Channel!

The Bot cannot read more than 1 CTP Listeners in one channel! It may be available in the future though!

Pre-Installed CTP Listeners
  1. Channel that broadcasts if a member joined the server - Welcome_Channel
  2. Channel that broadcasts if a member left the server - Leave_Channel
  3. Channel that broadcasts if the bot has an upcoming update - Axeltov_Bot_Update
  4. Channel that broadcasts if a member received a Global Ban - Global_Bans
  5. Channel that logs moderator's actions. This includes warns, bans and kicks with reasons. - Mod_Logs Not available

Encountered any problems during the installation?

Did the bot provided back the server information invalid? Are one of the CTP Listener's not working as it should? Please contact the Support Discord as they will definitely provide assistance!
